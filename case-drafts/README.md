# Case drafts

Рабочая папка для пересборки кейсов портфолио. Когда `final.md` готов — переносим в `src/data/projects.ts` и удаляем драфт (или оставляем как архив).

## Структура

Каждый кейс — это `<slug>/` с тремя файлами:

- `interview.md` — сырая запись интервью (Claude добавляет вопросы, Егор отвечает)
- `shopping-list.md` — список того, что нужно собрать: визуалы, цифры, ссылки
- `final.md` — чистовой текст под формат `Project`

## Шаблон кейса

См. `memory/portfolio-case-template.md` (auto-memory Claude).

Кратко:
1. Хук (1 строка + цифра)
2. Контекст
3. Вызов
4. Что я делал — три уровня: команда/процессы, продукт/арт-дирекшн, артефакт
5. Визуалы (с подсказкой формата)
6. Цифры (с уровнем уверенности)
7. Результат (3–4 буллета)
8. Пруфы / ссылки
